# 성장 기회 정의 — 닥터나우 vs 나만의닥터

> WebScout 크롤링 데이터 기반, Growth Marketing 관점 (2026-03-12)

---

## 닥터나우 — 기회 영역

### 기회 1: 웹 SEO 콘텐츠 부재

**현상:** 사이트맵에 31개 URL만 존재. 앱 내부에 콘텐츠가 집중되어 웹에서의 검색 유입이 극히 제한적.

| 지표 | 현재 | 기회 |
|---|---|---|
| 사이트맵 URL 수 | 31 pages | 수천 개로 확장 가능 |
| 검색 유입 키워드 | 브랜드명 위주 | 질환/증상/약물 롱테일 키워드 |
| 진료비 비교 페이지 | `/non-benefit` 5 pages | 병원·시술별 수천 페이지 |

**나만의닥터는 `/medical-cost`만 14,964 페이지로 가격 검색 SEO를 장악 중**

---

### 기회 2: 웹 전환 구조 약함

**현상:** 웹사이트가 앱 설치 유도용 게이트웨이에 그치고 있어, 웹에서의 직접적 가치 제공이 부족.

- 전환 페이지(`/naver/pay/register`)가 depth 3에 위치 → 접근성 낮음
- 퍼포먼스 마케팅용 전용 랜딩 페이지 부재
- 증상별/진료과별 전용 랜딩이 있지만 사이트맵에 미등록

---

### 기회 3: 웹 후기 노출 극대화

**현상:** 25만+ 후기와 4.8점이라는 강력한 사회적 증거를 보유하지만 검색엔진에서의 활용이 제한적.

| 항목 | 현재 | 기회 |
|---|---|---|
| 후기 노출 | 홈 캐러셀 (앱 리뷰 인용) | 후기 개별 페이지 → SEO 인덱싱 |
| 구조화 데이터 | 미확인 | Review, AggregateRating 스키마 |
| UGC SEO | 실시간 Q&A (앱 내) | 웹에 노출 → 롱테일 키워드 유입 |

---

## 나만의닥터 — 기회 영역

### 기회 1: 깊은 콘텐츠 접근성 문제

**현상:** 19,934페이지 중 4,884페이지가 depth 4+ → 검색엔진 크롤링·사용자 접근성 모두 저하.

| 지표 | 현재 | 기회 |
|---|---|---|
| depth 4+ 페이지 | 4,884개 (24.5%) | 내부 링크 구조 개선으로 depth 축소 |
| `/healthLab` 평균 depth | 4.0 | 2~3으로 줄이면 인덱싱 개선 |
| 파편화된 FAQ | 개별 URL이 매우 길고 깊음 | 카테고리 허브 페이지 구축 |

---

### 기회 2: 사회적 증거(후기) 미노출

**현상:** 50만명이 이용한다고 하지만, 웹에서 사용자 후기·리뷰가 전혀 보이지 않음.

| 항목 | 닥터나우 | 나만의닥터 |
|---|---|---|
| 웹 후기 노출 | 25만+ 캐러셀 | **X** |
| 실시간 Q&A | O | **X** |
| 앱스토어 리뷰 인용 | O | **X** |

닥터나우는 후기를 전면에 내세워 전환을 유도하는 반면, 나만의닥터는 콘텐츠 양으로 승부하되 신뢰 구축 장치가 부족.

---

### 기회 3: 전환 퍼널 불명확

**현상:** 대규모 콘텐츠로 유입은 있지만, 웹 → 앱 전환 경로가 약함.

- 명확한 CTA가 각 콘텐츠 페이지에 부족
- 전환 페이지 구조가 불명확 (`/find-color-game/start-loading` 이 전환 페이지?)
- 진료비 비교 → 바로 진료 예약 연결이 약함

## 양사 공통 기회

### 1. 구조화 데이터 (Schema.org)

두 사이트 모두 MedicalOrganization, FAQPage, MedicalWebPage, Review 등의 구조화 데이터 적용 여지가 큼. Google Health SERP 노출에 직결.

### 2. 웹앱 전환 (PWA)

"앱 다운로드 → 진료" 퍼널에서 이탈하는 사용자를 웹에서 직접 진료 접수까지 가능하게 하면 전환율 대폭 개선 가능.

### 3. 퍼포먼스 마케팅 랜딩 최적화

증상별/진료과별/시술별 전용 랜딩 페이지가 부족. A/B 테스트 가능한 랜딩 구조 필요.
