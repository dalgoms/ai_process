## 콘텐츠 전략 비교

### 닥터나우 — "앱 중심 게이트웨이"

```
홈 → 비대면 진료 카테고리 (/telemedicine)
홈 → 병원 찾기 (/hospitals)
홈 → 건강 정보 (/content → faq, magazine, qna)
홈 → 질환백과 (/disease-encyclopedia)
홈 → 의약품사전 (/medicine-dictionary)
홈 → 비급여 진료비 (/non-benefit)
```

**크롤링 결과 (31 pages):**

웹사이트 자체는 간결한 구조로, 실제 서비스 경험은 앱에 집중. 웹은 앱 설치를 유도하는 게이트웨이 역할.

**강점:**
- 홈페이지에 25만+ 후기 캐러셀 → 강력한 사회적 증거
- FAQ 섹션 직접 노출 → 전환 장벽 제거
- 실시간 의료 상담(Q&A) 기능 → UGC 축적 잠재력
- 진료과별 카테고리 구분 (소아과, 탈모, 여드름 등)

**약점:**
- 사이트맵에 31개 URL만 존재 → 웹 SEO 유입 한계
- 콘텐츠(매거진, 질환백과 등)가 앱 내부에 집중, 웹 노출 부족
- 검색엔진에서 개별 콘텐츠 페이지 인덱싱 어려움

### 나만의닥터 — "콘텐츠 SEO 머신 + 리텐션 엔진"

```
홈 → 비대면 진료
홈 → 닥터포인트 (진료비 할인)
홈 → 닥터캐시 (건강관리 리워드)
홈 → 최저가 병원 예약 (/medical-cost → 14,964 pages!)
홈 → 건강연구소 (/healthLab → FAQ 4,887 pages!)
홈 → 일요일/야간 병원 찾기
```

**크롤링 결과 (19,934 pages):**

웹에서 대규모 콘텐츠를 통해 검색 유입을 확보하는 전략. 특히 진료비 비교(/medical-cost)가 압도적.

**강점:**
- `/medical-cost` 14,964 페이지 → 진료비 검색 키워드 대량 확보
- `/healthLab` 4,887 페이지 → 건강 FAQ, 질환 정보 대량 콘텐츠
- 포인트/캐시 시스템으로 사용자 락인
- "건강관리만 해도 리워드" → 비진료 시점에도 앱 사용 유도
- 최저가 비교 → 가격 투명성으로 신뢰 구축

**약점:**
- depth 4+ 페이지가 4,884개 → 깊은 콘텐츠 접근성 문제
- 웹에서 직접 진료 불가 (앱 필수)
- 홈페이지에 사용 후기 미노출 (사회적 증거 약함)

|---|---|---|
| 인덱싱 가능 페이지 | 31 (사이트맵 기준) | 19,934 (사이트맵 기준) |
| 콘텐츠 볼륨 | 소량 (앱 내부 집중) | **대량** (진료비 15K + 건강정보 5K) |
| 키워드 커버리지 | 좁음 (브랜드 + 서비스명) | **넓음** (진료비, 증상, 질환, 병원별) |
| 진료과별 랜딩 | O (카테고리 분류) | X (통합 구조, 진료비 페이지로 대체) |
| UGC | 실시간 의료상담 (앱 내) | X |
| 가격 비교 콘텐츠 | `/non-benefit` (5 pages) | `/medical-cost` (**14,964 pages**) |

나만의닥터가 SEO 콘텐츠 양에서 압도적 우위. "OO 진료비", "OO 병원 가격" 같은 롱테일 키워드를 대량으로 커버하는 전략. 닥터나우는 웹 콘텐츠 양은 적지만, 앱 내 콘텐츠와 사회적 증거(후기)에서 강점.
