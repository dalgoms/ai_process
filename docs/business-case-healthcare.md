# 비즈니스 케이스: 헬스케어 플랫폼 (나만의닥터 사례)

비대면 진료 플랫폼처럼 B2C(환자) + B2B(병원/약국 파트너) 투 사이드 마켓에
AI 업무 자동화 파이프라인을 적용하는 방법.

---

## 적용 구조 요약

```
┌─────────────────────────────────────────────────┐
│              B2B (병원/약국 파트너)                │
│                                                  │
│  제휴 문의 폼 → Make.com → Notion Leads DB       │
│                          → Telegram 알림          │
│                          → 자동 응답 이메일        │
│  파트너 성과 → Schedule → 주간 리포트 발송         │
│  팔로업 → 자동 리마인더 (3일 내 미연락 시)         │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│              B2C (환자/사용자)                     │
│                                                  │
│  고객 문의 → Make.com → AI 자동 분류 → 담당자 라우팅│
│  콘텐츠 → Content Pipeline → AI 초안 → 검토 알림  │
│  약 배송 → 상태 변경 시 → 환자 알림               │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│              내부 (개발/운영)                      │
│                                                  │
│  버그 리포트 → Notion → GitHub Issue → AI 코드 수정│
│  규제 변경 → 크롤링 → AI 영향 분석 → 법무팀 알림  │
└─────────────────────────────────────────────────┘
```

## B2B: 병원/약국 파트너 영업

### 1. 병원 제휴 문의 자동화

**흐름:**
```
병원장/약사가 제휴 문의 폼 작성
    → Make.com Webhook
    → Notion Leads DB 저장 (Type: b2b, Source: partner-page)
    → Telegram "신규 병원 제휴 문의" 알림
    → 자동 응답 이메일 "제휴 검토 후 영업담당이 연락드리겠습니다"
```

**커스텀 Leads DB 필드:**

| 필드 | 타입 | 용도 |
|------|------|------|
| 병원명 | text | 의원/약국 이름 |
| 진료 과목 | select | 내과, 피부과, 소아과, 정신건강의학과 등 |
| 소재지 | text | 지역 (서울 강남, 경기 성남 등) |
| 의사 수 | number | 병원 규모 파악 |
| 월 진료 건수 | number | 리드 스코어링 기준 |
| 기존 비대면 진료 경험 | checkbox | 영업 전략 분류 |

### 2. 파트너 팔로업 자동 리마인더

```
제휴 문의 후 3일 내 연락 안 했으면
    → Telegram "팔로업 필요: OO병원 (피부과, 강남)"
    → 영업담당에게 이메일 알림
```

### 3. 파트너 병원 주간 성과 리포트

```
Schedule (매주 월요일 09:00)
    → Notion에서 파트너 병원 데이터 조회
    → 주간 요약 Telegram/이메일 발송
```

리포트 예시:
```
[주간 파트너 리포트]
신규 파트너: 5곳
활성 파트너: 120곳
이탈 위험: 3곳 (30일 이상 비활성)
이번 주 총 진료 건수: 2,340건
```

## B2C: 환자/사용자

### 4. 고객 문의 AI 자동 분류

```
앱/웹 문의 → Make.com
    → OpenAI 키워드 분석
    → 자동 분류 + 카테고리별 담당자 라우팅
    → Notion에 저장 + 긴급도별 알림
```

**분류 기준:**

| 키워드 | 분류 | 긴급도 | 라우팅 |
|--------|------|--------|--------|
| "환불", "결제 오류" | 결제 이슈 | 높음 | CS팀 즉시 |
| "의사가", "진료 불만" | 서비스 불만 | 높음 | QA팀 |
| "사용법", "어떻게" | 일반 문의 | 보통 | FAQ 자동응답 |
| "제휴", "협력" | B2B 문의 | 보통 | 영업팀 |
| "처방", "약 배송" | 배송 문의 | 보통 | 물류팀 |

### 5. 콘텐츠 마케팅 자동화

```
마케터가 Content Pipeline DB에 아이디어 등록
    → OpenAI가 초안 자동 생성
    → Notion Draft 필드에 저장
    → Status: review + Telegram "검토 필요" 알림
    → 마케터 검토 후 Approved → 게시
```

**시즌별 자동 콘텐츠 예시:**

| 시즌 | 콘텐츠 주제 |
|------|-----------|
| 봄 (3~5월) | 알레르기/비염 비대면 진료 가이드 |
| 여름 (6~8월) | 피부 트러블, 여름 감기 관리 |
| 가을 (9~11월) | 독감 백신 예약 안내, 건강검진 |
| 겨울 (12~2월) | 감기/코로나 비대면 진료, 면역력 관리 |

### 6. 약 배송 상태 알림 (확장)

```
약 배송 시작 → Make.com → Telegram/카카오 알림
"처방약이 발송되었습니다. 예상 도착: 오늘 18:00"

배송 완료 → Make.com → 만족도 조사 링크 발송
```

## 내부 개발팀 자동화

### 7. 앱 버그 리포트 → AI 자동 코드 수정

```
CS팀이 Notion Work Inbox에 버그 등록
    "iOS 앱에서 결제 완료 후 화면 멈춤"
    → ai_process notion-sync가 감지
    → GitHub Issue 자동 생성 (ai-task 라벨)
    → AI Codex가 코드 분석 + 수정 PR 생성
    → CI 검증 → 개발자 승인 → 배포
```

### 8. 의료 데이터 규제 문서 관리

```
규제 변경 감지 (뉴스 크롤링 또는 수동 등록)
    → Notion에 저장
    → AI가 영향 범위 분석 초안 작성
    → 법무팀에 Telegram 검토 알림
```

## 도입 효과 예측

| 영역 | Before | After | 영향 |
|------|--------|-------|------|
| 병원 제휴 문의 응답 | 이메일 확인 후 1~2일 | 즉시 알림 + 자동 응답 | 제휴 전환율 상승 |
| 파트너 팔로업 | 수동 관리, 누락 빈번 | 자동 리마인더 | 이탈률 감소 |
| 고객 문의 처리 | CS팀 수동 분류 | AI 자동 분류 + 라우팅 | 처리 속도 50%+ 향상 |
| 콘텐츠 생산 | 마케터 직접 작성 1~2시간/건 | AI 초안 + 사람 검토 15분/건 | 생산량 3~5배 |
| 버그 수정 | 개발자 직접 코딩 | AI 자동 수정 + 검증 | 핫픽스 시간 80% 단축 |
