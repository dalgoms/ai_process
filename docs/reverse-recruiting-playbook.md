# 역방향 리크루팅 플레이북

> 지원하지 않는다. 진단한다. 회사가 찾아오게 만든다.

---

## 컨셉

일반적인 구직: **이력서 제출 → 면접 대기 → 합격 기다림**

역방향 리크루팅: **회사 분석 → 진단 공유 → 회사가 먼저 연락**

"살짝 살펴봤는데요" 하고 던진 분석이 12개 문서에 SWOT, KPI, 90일 로드맵까지 담겨 있으면 — 면접 전에 이미 이긴 겁니다.

---

## 프로세스

```
1. 회사 선정 (30분)
   │
2. WebScout 크롤링 (30분)
   │
3. case-template 복사 → 분석 작성 (2~3시간)
   │
4. GitHub push
   │
5. 링크 전달 or LinkedIn 공유 (10분)
```

### Step 1: 회사 선정

관심 기준:
- 성장 중이지만 마케팅/그로스 구조가 아직 정립되지 않은 곳
- SEO 콘텐츠가 많지만 전환 구조가 약한 곳
- 앱/웹 서비스가 있어 WebScout으로 분석 가능한 곳
- 본인의 경험(퍼포먼스 마케팅, CRM, 모바일 UX)이 직접 기여할 수 있는 곳

### Step 2: WebScout 크롤링

```
https://webscout-next-8veo.vercel.app/
```

URL 입력 → 사이트 구조 분석 결과 확인:
- 전체 페이지 수, depth 프로파일
- 주요 섹션/클러스터
- 전환 페이지 유무
- 캠페인/UTM URL
- 숨겨진 페이지

### Step 3: 분석 작성

`case-template/` 폴더를 복사하여 회사명으로 rename.

**최소 작성 문서 (3개):**

| 문서 | 내용 | 소요 |
|---|---|---|
| `service-structure-analysis.md` | WebScout 크롤링 기반 구조 인사이트 | 1시간 |
| `problem.md` | 현 구조에서 보이는 성장 기회 | 30분 |
| `solution.md` | 방향성 제안 + 수익 연결 | 30분 |

**풀 버전 추가 (선택):**

| 문서 | 내용 |
|---|---|
| `competitive-landscape.md` | 경쟁 환경 + SWOT |
| `success-metrics.md` | KPI 프레임워크 |
| `growth-marketing-role.md` | 기여 영역 + 90일 목표 |

### Step 4: GitHub push

```bash
cp -r case-template/ [company-name]/
# 문서 작성 후
git add -A
git commit -m "Add service diagnostic: [company-name]"
git push origin master
```

ai_process README.md의 Service Diagnostics 테이블에 행 추가.

### Step 5: 공유

**직접 전달 (입사 제안/관심 있는 곳):**

> 안녕하세요, 이세영입니다.
> 서비스가 흥미로워서 간단하게 살펴본 내용 정리해봤습니다.
> [GitHub 링크]
> 참고되시면 좋겠습니다.

**LinkedIn 공유 (퍼블릭 포지셔닝):**

> [회사/산업] 서비스 구조를 살펴보면서 몇 가지 흥미로운 패턴을 발견했습니다.
>
> [핵심 인사이트 1~2줄]
>
> 상세 분석: [GitHub 링크]
>
> #GrowthMarketing #ServiceDiagnostics #CRO

## 타겟 산업별 분석 프레임워크

| 산업 | 주요 분석 포인트 | 솔루션 방향 |
|---|---|---|
| **헬스케어** | SEO 콘텐츠 유입 → 전환 경로, 앱 리텐션 | 참여 레이어, CRM, 전환 특화 랜딩 |
| **이커머스** | 상품 페이지 구조, 퍼널 이탈, 리타겟팅 | CRO, 개인화 추천, 리텐션 캠페인 |
| **SaaS** | 온보딩 퍼널, 프리미엄 전환, 이탈 방지 | 퍼널 최적화, 인앱 넛지, CRM |
| **교육/콘텐츠** | 콘텐츠 소비 패턴, 유료 전환 | 콘텐츠 개인화, 구독 구조 |
| **핀테크** | 가입 퍼널, KYC 이탈, 활성 사용 | 온보딩 간소화, 참여 루프 |
