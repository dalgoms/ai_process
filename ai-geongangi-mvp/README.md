# AI 건강 비서 — 서비스 분석 노트

> 나만의닥터 앱과 웹사이트를 살펴보면서 정리한 생각들입니다.

---

## 배경

나만의닥터에 관심을 갖게 되면서, 앱과 웹사이트 구조를 직접 살펴보았습니다. 직접 만든 크롤링 도구([WebScout](https://github.com/dalgoms/webscout-next))로 사이트 구조를 분석하고, 헬스케어 앱 시장을 조사하면서 몇 가지 생각을 정리해 보았습니다.

나만의닥터는 **2만 개 이상의 건강 콘텐츠 페이지**로 검색 유입을 이미 확보하고 있고, 이 강력한 유입 엔진 위에 **일상 참여 레이어**를 얹으면 더 큰 성장이 가능하겠다는 생각이 들었습니다. 그 방향성을 "AI 건강 비서"라는 컨셉으로 풀어보았습니다.

## AI 건강 비서 컨셉

아플 때만 쓰는 앱이 아니라, 매일 함께하는 건강 파트너.

사용자의 건강 행동을 분석하여 **매일 건강 인사이트**, **스마트 알림**, **건강 타임라인**을 제공합니다. 앱이 사용자를 찾아가는 구조 — 매일 아침과 저녁, 개인화된 건강 메시지와 함께.

```
현재:    SEO 유입  →  콘텐츠 열람  →  이탈 (일회성)
아이디어: SEO 유입  →  콘텐츠 열람  →  AI 건강 비서 온보딩  →  일상 참여  →  비대면 진료/약 배달 전환
```

## 주요 기능 아이디어

| 기능 | 설명 | 나만의닥터 연결 |
|---|---|---|
| **스마트 건강 알림** | 수면, 걸음 수, 행동 패턴에 대한 AI 기반 일일 인사이트 | 이상 징후 감지 시 비대면 진료 연결 |
| **건강 챗봇** | 증상, 약물, 건강 조언에 대해 자연어로 질문 | 심화 상담 필요 시 의사 연결 |
| **건강 타임라인** | 모든 건강 활동과 데이터를 달력 형태로 조회 | 진료 이력 + 처방 이력 통합 |
