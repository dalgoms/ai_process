# 서비스 구조 관찰 (Service Structure Observation)

> "나만의닥터" 웹사이트 구조를 분석하여 도출한 인사이트입니다.
> 분석은 간단한 내부 크롤링 도구인 **WebScout**을 사용하여 수행되었습니다.

---

## 전체 규모

크롤링 결과, 약 **20,000개의 내부 페이지**가 감지되었습니다.

일반적인 헬스케어 서비스 앱 대비 매우 큰 규모이며, 이는 단순한 서비스 페이지가 아닌 대량의 콘텐츠 페이지가 존재함을 의미합니다.

## 주요 콘텐츠 클러스터

특히 두 개의 대규모 SEO 콘텐츠 클러스터가 확인되었습니다:

| URL 경로 | 추정 역할 |
|---|---|
| `/medical-cost` | 의료비 정보 콘텐츠 |
| `/healthLab` | 건강 정보 / 건강 연구소 콘텐츠 |

이 클러스터들은 각각 수천 개 단위의 하위 페이지를 포함하고 있으며, 검색 엔진을 통한 유입을 목적으로 설계된 것으로 보입니다.

## SEO 중심 유입 전략

이 구조에서 읽을 수 있는 사용자 유입 흐름은 다음과 같습니다:

```
검색 (Google/Naver)  →  의료 정보 콘텐츠  →  상담 / 서비스 이용
```

사용자가 건강 관련 키워드를 검색하면, 나만의닥터의 콘텐츠 페이지에 도달하고, 그곳에서 서비스로 전환되는 구조입니다.

이는 **SEO 기반 사용자 획득 전략**이 서비스의 핵심 성장 동력임을 시사합니다.

## Navigation Depth 문제

그러나 한 가지 주목할 점이 있습니다.

상당수의 페이지가 홈페이지에서 **3 클릭 이상 깊은 곳**에 위치해 있었습니다. 일부 페이지는 일반적인 navigation으로는 도달하기 어렵거나, 사실상 숨겨진 상태로 보였습니다.

이는 두 가지 문제를 야기할 수 있습니다:

- **검색 엔진 크롤링 효율 저하** — 깊은 페이지는 검색 엔진이 발견하고 색인하기 어려울 수 있습니다.
- **전환 기회 감소** — 가치 있는 콘텐츠가 깊이 묻혀 있으면, 사용자가 해당 페이지에 도달하더라도 서비스 전환으로 이어지기 어렵습니다.

## 웹 구조 분석 (Web Structure Observation)

WebScout 크롤링을 통해 웹사이트의 페이지 구성을 추가로 분석했습니다.

### 페이지 구성 현황

| 항목 | 관찰 결과 |
|---|---|
| 전체 페이지 수 | 약 **20,000개** |
| 내부 콘텐츠 페이지 | 거의 전부 — 대다수가 건강 정보 콘텐츠 |
| 캠페인 랜딩 페이지 | **극소수** 감지 |
| 주요 콘텐츠 클러스터 | `/healthLab` (건강 정보 대규모 클러스터) |

20,000개에 달하는 페이지 중 거의 대부분이 내부 콘텐츠 페이지입니다. 별도의 캠페인 랜딩 페이지는 거의 발견되지 않았습니다.

### SEO 중심 유입 구조

이 구성은 명확한 SEO 기반 사용자 획득 전략을 보여줍니다.

```
검색 (Google/Naver)  →  건강 정보 콘텐츠 (healthLab 등)  →  서비스 이용 / 상담
```

대량의 건강 정보 콘텐츠를 통해 검색 유입을 확보하고, 콘텐츠를 읽은 사용자가 자연스럽게 서비스로 전환되는 흐름입니다.

콘텐츠 볼륨 자체가 곧 유입 채널인 구조로, organic search에 크게 의존하고 있는 것으로 보입니다.

### 퍼포먼스 마케팅 랜딩 페이지의 기회

주목할 점은 퍼포먼스 마케팅용 랜딩 페이지가 거의 감지되지 않았다는 것입니다.

일반적으로 유료 광고(Google Ads, Meta Ads 등)를 통한 유입을 극대화하려면, 특정 캠페인에 최적화된 전용 랜딩 페이지가 필요합니다. 이 페이지들은 보통 다음과 같은 특성을 가집니다:

- 명확한 단일 CTA (Call to Action)
- 광고 메시지와 일치하는 헤드라인
- 전환에 집중된 간결한 구조

이 영역은 나만의닥터에게 **전환율 최적화(CRO)의 확장 기회**가 될 수 있습니다.

SEO 콘텐츠로 유입된 사용자를 서비스로 전환하는 중간 단계 — 즉, 전환 특화 랜딩 페이지 — 를 보강하면 유입 대비 전환율을 크게 개선할 수 있을 것입니다.

### SEO 기반 콘텐츠 구조 관찰

크롤링된 페이지의 상당수가 `/healthLab` 하위의 건강 정보 콘텐츠 허브에 속해 있습니다. 질환 정보, 건강 상식, 의료 가이드 등 다양한 주제가 개별 페이지로 구성되어 있으며, 이들이 전체 페이지의 핵심 볼륨을 차지합니다.

이 구조는 사용자가 **건강 관련 검색을 통해 콘텐츠에 진입**하고, 정보를 소비한 뒤 서비스로 넘어가는 SEO 기반 유입 전략을 명확하게 보여줍니다.

그러나 현재 구조에서는 콘텐츠 소비 이후의 경험이 끊깁니다. 사용자가 정보를 읽고 떠나면 재방문의 이유가 없습니다.

여기에 **AI 건강 비서의 개인화된 건강 인사이트**를 연결할 수 있습니다.

```
기존:    검색 유입  →  건강 정보 열람  →  이탈
개선:    검색 유입  →  건강 정보 열람  →  AI 건강 비서 온보딩  →  지속적 제품 참여
```

정보성 트래픽으로 들어온 사용자에게 AI 건강 비서가 "당신의 건강 데이터를 기반으로 매일 인사이트를 드립니다"라는 가치를 제안하면, 일회성 방문을 **지속적인 제품 참여**로 전환할 수 있습니다.

SEO 콘텐츠가 유입의 입구라면, AI 건강 비서는 그 유입을 붙잡아 두는 장치가 됩니다.

### 참여형 콘텐츠 구조 관찰

정보성 콘텐츠 외에, 크롤링 과정에서 **인터랙티브 참여형 페이지**도 발견되었습니다.

대표적으로 `/find-color-game`과 같은 간단한 게임 형태의 페이지가 존재합니다. 정적인 건강 정보 페이지와는 성격이 다른, 사용자 참여를 유도하는 콘텐츠입니다.

이는 나만의닥터가 이미 **인터랙티브 참여 메커니즘을 실험**하고 있음을 보여줍니다.

이러한 참여형 페이지의 역할은 다음과 같이 추정됩니다:

- **사용자 참여 도구** — 가볍게 즐길 수 있는 경험으로 체류 시간 증가
- **바이럴 공유 메커니즘** — SNS 공유를 통한 자연스러운 유입 확대
- **경량 사용자 획득 실험** — 저비용으로 신규 유입 채널을 테스트

여기서 AI 건강 비서와의 연결 가능성이 보입니다.

현재의 참여형 콘텐츠가 일회성 재미에 그친다면, AI 건강 비서는 이를 **일상적인 건강 인터랙션**으로 발전시킬 수 있습니다.

```
현재:    게임/퀴즈  →  한 번 즐기고 이탈
확장:    건강 퀴즈 / 건강 챌린지  →  AI 건강 비서 연결  →  매일 건강 인터랙션
```

인터랙티브 참여 + AI 건강 동반자를 결합하면, 가벼운 재미에서 시작해 지속적인 건강 관리 습관으로 이어지는 경험을 설계할 수 있습니다.

### 콘텐츠 기반 유입 구조와 확장 가능성

`/healthLab` 구조는 단순한 콘텐츠 모음이 아닙니다. 이것은 검색 엔진을 통해 사용자를 플랫폼으로 끌어들이는 **SEO 기반 유입 엔진**입니다.

사용자는 대부분 증상 검색이나 의료 정보 질의를 통해 플랫폼에 처음 도달하는 것으로 추정됩니다.

```
"두통 원인"  →  healthLab 콘텐츠  →  플랫폼 진입
"혈압약 부작용"  →  healthLab 콘텐츠  →  플랫폼 진입
```

이 전략은 사용자 획득에는 효과적입니다. 검색 의도가 명확한 사용자를 대량으로 유입시킬 수 있기 때문입니다.

그리고 **획득 이후**에 다음 기회가 있습니다.

정보를 얻은 사용자는 목적을 달성하면 떠납니다. 콘텐츠를 읽는 것과 서비스를 지속적으로 사용하는 것은 전혀 다른 행동입니다. 정보성 방문자를 지속 사용자로 전환하는 경험이 부재하면, 아무리 많은 트래픽도 일회성에 그칩니다.

여기서 다음 기회가 열립니다: **개인화된 건강 관리 경험**을 통한 전환.

AI 건강 비서는 정보 소비와 일상 건강 참여 사이의 **다리 역할**을 할 수 있습니다.

```
정보 소비 단계          →  전환 단계              →  지속 참여 단계
───────────────────     ──────────────────       ──────────────────
"두통 원인" 검색         AI 건강 비서 제안            매일 건강 인사이트
healthLab 콘텐츠 열람    "수면 패턴을 분석해       아침/저녁 건강 루프
정보 획득                드릴까요?"                건강 타임라인 구축
```

사용자가 증상을 검색하여 콘텐츠를 읽은 바로 그 시점에, AI 건강 비서가 "당신의 건강 패턴을 분석해 매일 인사이트를 드립니다"라고 제안합니다. 단발성 정보 탐색이 **지속적인 건강 동반자 관계**의 시작점이 됩니다.

콘텐츠가 문을 열고, AI 건강 비서가 그 안에 머물게 하는 구조입니다.

## 시사점

나만의닥터의 구조는 SEO를 통한 대규모 유입에는 효과적이지만, 세 가지 개선 여지가 있습니다:

1. **콘텐츠 발견성(discoverability)** — navigation depth가 깊어 가치 있는 페이지가 묻힐 수 있음
2. **전환 경로 최적화** — 퍼포먼스 마케팅 랜딩 페이지 부재로 유료 유입 전환에 한계
3. **정보성 트래픽의 제품 전환** — SEO 콘텐츠 유입 후 지속 참여로 이어지는 장치 부재

## 실행 제안: 합류하게 된다면

이 분석을 바탕으로 세 가지 실행 방향을 제안합니다:

### 1. healthLab 콘텐츠 → AI 건강 비서 온보딩 브릿지

2만 개 콘텐츠 페이지 하단에 AI 건강 비서 온보딩 CTA를 삽입합니다. "이 증상이 반복된다면, AI 건강 비서가 패턴을 분석해 드립니다" — SEO 유입을 제품 참여로 전환하는 가장 빠른 방법입니다.

SORIZAVA에서 /livesteno 앵커 페이지로 91% 반송률 환경에서 +1,078% 전환을 만들어낸 경험을 직접 적용합니다.

### 2. 퍼포먼스 랜딩 페이지 신규 구축

현재 부재한 캠페인 전용 랜딩 페이지를 설계합니다. Google Ads, Meta Ads 등 유료 채널별로 최적화된 전환 페이지를 구축하여, SEO 의존도를 낮추고 유료 유입의 ROI를 극대화합니다.

8~15억 규모 광고 예산을 운용하며 매체별 랜딩 최적화를 해온 노하우를 이식합니다.

### 3. Navigation depth 개선 + 내부 링크 구조 재설계

3클릭 이상 깊이에 묻혀 있는 가치 콘텐츠의 접근성을 개선합니다. WebScout으로 크롤링한 IA 데이터를 기반으로 내부 링크 구조를 재설계하고, 핵심 전환 페이지로의 경로를 단축합니다.
