# AI 알림 — 아침/저녁 건강 루프

## 컨셉

AI 건강 비서는 **하루 두 번의 터치포인트**로 사용자가 건강에 관심을 유지하도록 합니다. 부담스럽지 않게. 아침에 한 번, 저녁에 한 번. 각 알림은 AI가 생성하며, 개인화되고, 실행 가능합니다.

## 루프 구조

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   ☀ 아침                      🌙 저녁           │
│   건강 인사이트                행동 리플렉션      │
│                                                 │
│   "오늘 집중할                  "오늘 하루가      │
│    것은 이것입니다."             이랬습니다."      │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 아침: 건강 인사이트

하루의 시작에 전달됩니다. 아침 알림은 최근 데이터를 기반으로 **미래 지향적 인사이트**를 제공합니다.

**예시:**

- "어젯밤 7.2시간 수면하셨습니다 — 좋은 회복이네요. 오늘 에너지가 좋을 겁니다."
- "이번 주 걸음 수가 줄어들고 있습니다. 점심 전에 산책을 해보세요."
- "알림: 이틀 후 약 리필 예정입니다."
- "어제 혈압이 약간 높았습니다. 수분을 충분히 섭취하고 스트레스를 줄여보세요."

**목표:** 인식으로 하루를 시작합니다. 사용자에게 하나의 생각거리를 줍니다.

## 저녁: 행동 리플렉션

저녁에 전달됩니다. 저녁 알림은 그날의 건강 행동에 대한 **회고적 요약**을 제공합니다.

**예시:**

- "오늘 9,400보를 걸으셨습니다 — 주간 평균 이상이네요. 잘하셨어요."
- "3끼 식사를 기록하셨고 칼로리 목표 안에 들어왔습니다."
- "오늘 수분 섭취를 기록하지 않으셨습니다. 수분은 수면 질에 영향을 줍니다."
- "안정 시 심박수가 평소보다 낮았습니다 — 좋은 회복의 신호입니다."

**목표:** 되돌아보며 하루를 마무리합니다. 좋은 습관을 강화하고, 부드럽게 부족한 부분을 알려줍니다.

## 왜 두 번인가?

| 접근 방식 | 문제점 |
|---|---|
| 알림 없음 | 사용자가 앱의 존재를 잊는다 |
| 알림이 너무 많음 | 일주일 내에 알림을 끈다 |
| 일반적인 알림 | 무시한다 — 개인적이지 않으니까 |
| **AI 개인화된 두 번의 터치포인트** | **사용자가 일상적인 건강 인식 습관을 만든다** |

아침/저녁 루프는 **딱 적절한 양**으로 설계되었습니다 — 존재감은 있되 부담스럽지 않고, 유용하되 시끄럽지 않게.

---

*루프가 리듬을 만듭니다: 의도로 시작하고, 되돌아보며 마무리합니다. 시간이 지나면 이것이 사용자의 일상 건강 습관이 됩니다.*
