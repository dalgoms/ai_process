# CRM 연동 — (기업 내 이용중인 솔루션)

## CRM이 필요한 이유

AI 건강 비서는 앱 내에서 인사이트를 생성하지만, 진정한 참여를 이끌어내려면 사용자가 이미 사용하는 채널 — 푸시 알림, 이메일, 인앱 메시지 — 을 통해 전달되어야 합니다. 여기서 CRM이 등장합니다.

Braze는 AI 건강 비서의 건강 커뮤니케이션을 위한 **전달 및 오케스트레이션 레이어** 역할을 합니다.

## 아키텍처

```
AI 건강 비서 앱  →  사용자 이벤트  →  CRM  →  푸시 / 이메일 / 인앱
                                    ↓
                              세그멘테이션
                              A/B 테스팅
                              캠페인 자동화
```

## 사용자 이벤트

AI 건강 비서에서의 모든 의미 있는 사용자 행동은 이벤트로 추적되어 Braze로 전송됩니다. 이 이벤트들이 세그멘테이션, 타겟팅, 캠페인 트리거를 구동합니다.

**이벤트 예시:**

| 이벤트명 | 설명 | 트리거 |
|---|---|---|
| `health_insight_viewed` | 사용자가 아침/저녁 인사이트를 열람 | 알림 탭 |
| `chatbot_question_asked` | 사용자가 챗봇에서 질문 | 채팅 전송 |
| `timeline_viewed` | 사용자가 건강 타임라인 열람 | 화면 조회 |
| `medication_logged` | 사용자가 약물 복용 기록 | 수동 기록 |
| `step_goal_achieved` | 사용자가 일일 걸음 수 목표 달성 | 자동 감지 |
| `notification_dismissed` | 사용자가 건강 알림 닫음 | 스와이프 |
| `weekly_streak_completed` | 사용자가 7일 연속 참여 | 자동 감지 |

## 사용자 속성

이벤트 외에도 사용자 레벨 속성이 세그멘테이션을 위해 CRM에 동기화됩니다:

| 속성 | 예시 |
|---|---|
| `avg_daily_steps` | 6,500 |
| `sleep_avg_hours` | 6.8 |
| `active_days_last_7` | 5 |
| `medication_adherence` | 85% |
| `last_chatbot_use` | 2026-03-09 |

## 캠페인 예시

CRM을 활용하여 AI 건강 비서 사용자 행동 기반의 자동화 캠페인을 구축할 수 있습니다:

**리인게이지먼트:**
- 트리거: 사용자가 3일간 앱을 열지 않음
- 액션: 푸시 전송 — "AI 건강 비서에 새로운 인사이트가 기다리고 있어요."

**연속 기록 축하:**
- 트리거: `weekly_streak_completed` 이벤트
- 액션: 인앱 메시지 — "7일 연속 달성! 건강 인식이 쌓이고 있어요."

**약물 복용 리마인더 강화:**
- 트리거: `medication_adherence`가 70% 이하로 하락
- 액션: 푸시 전송 — "최근 기록을 놓친 것 같아요. 리마인더 설정을 도와드릴까요?"

**챗봇 참여 유도:**
- 트리거: 사용자가 챗봇을 한 번도 사용하지 않음
- 액션: 인앱 카드 — "AI 건강 비서에게 증상에 대해 물어볼 수 있다는 걸 아셨나요?"

## 나만의닥터 서비스 연결 캠페인

기존 캠페인에 더해, 나만의닥터의 핵심 수익원과 직접 연결되는 캠페인을 설계합니다:

**비대면 진료 전환:**
- 트리거: AI가 건강 이상 패턴을 감지 (예: 혈압 3일 연속 상승)
- 액션: 인앱 카드 — "혈압이 3일째 올라가고 있어요. 의사와 상담해볼까요?" + [비대면 진료 예약] CTA
- 측정: `health_alert_shown` → `telemedicine_booked` 전환율

**약 배달 리필:**
- 트리거: 약물 복용 기록 기반 리필 시점 예측 (잔여 5일)
- 액션: 푸시 전송 — "처방약이 5일 뒤 소진 예정이에요." + [리필 주문] CTA
- 측정: `refill_reminder_sent` → `delivery_ordered` 전환율

**healthLab 콘텐츠 재소비:**
- 트리거: 사용자의 건강 타임라인 데이터와 매칭되는 콘텐츠 발견
- 액션: 아침 인사이트에 관련 콘텐츠 링크 포함 — "수면이 부족한 요즘, 이 글이 도움이 될 수 있어요"
- 측정: `content_recommended` → `content_clicked` → `content_read` 퍼널

## 기대 효과

- **대규모 개인화** — Braze가 인구통계가 아닌 행동 기반으로 사용자를 세그먼트
- **A/B 테스트 가능** — 모든 메시지의 타이밍, 문구, 형식을 테스트 가능
- **측정 가능** — 알림 발송 → 열람 → 행동까지 전체 퍼널 추적
- **유연함** — 앱 업데이트 없이 캠페인 로직 변경 가능
- **수익 직결** — AI 참여 → 진료/약 배달 전환으로 CAC 대비 LTV 극대화

## SORIZAVA에서의 적용 경험

이 CRM 캠페인 설계는 이론이 아닙니다. SORIZAVA에서 실제로 구축하고 운영한 경험에 기반합니다:

- 8개 프로퍼티의 리드 파이프라인을 **자동화 오토메이션**으로 구축
- 고객 세그먼트별 서비스 소개서 **자동 발송** 설계
- 각 부서장에게 배포하여 **자율 운영 체계** 구축
- 전사 리드 통합 모니터링 + 개별 부서 분산 대응 프로세스 설계

동일한 설계 역량을 나만의닥터 CRM 캠페인에 적용합니다.
