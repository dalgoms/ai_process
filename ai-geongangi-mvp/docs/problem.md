# 문제 정의

## 헬스케어 앱은 아플 때만 쓴다

오늘날 헬스케어 앱의 사용 패턴은 단순합니다: 아프다 → 앱을 연다 → 기능을 쓴다 → 떠난다. 다음 문제가 생길 때까지 돌아올 이유가 없습니다.

이로 인해 근본적인 참여 격차가 생깁니다. 앱은 질병이나 걱정의 순간에만 가치를 가지며, 실제로 건강 습관이 형성되는 나머지 95%의 시간에는 아무런 역할을 하지 못합니다.

## 낮은 일일 참여도

대부분의 건강 앱은 첫 주 이후 급격한 이탈을 경험합니다. 사용자는 좋은 의도로 다운로드하여 몇 끼 식사나 운동을 기록하다가 곧 멈춥니다. 푸시 알림은 일반적이고, 대시보드는 복잡합니다. 건강한 평범한 날에 앱을 열어야 할 이유가 없습니다.

**결과:** 건강 앱은 "설치하고 잊는" 도구가 되어 리텐션이 매우 낮습니다.

## 기능 중심 설계, 사용자 중심이 아닌

현재 앱들은 기능 위주로 설계되어 있습니다:

- 만보기
- 약 복용 리마인더
- 증상 체커
- 칼로리 트래커

각 기능은 독립적으로 작동합니다. 사용자가 직접 여러 도구를 오가며 정보를 연결해야 합니다. "오늘 내 건강은 어때?"라는 통합된 뷰는 존재하지 않습니다 — 그저 유틸리티의 모음일 뿐입니다.

**부족한 것:** 이 모든 데이터 위에 앉아서 사용자에게 매일 의미 있고, 개인적이고, 시의적절한 건강 정보를 전달하는 레이어입니다.
