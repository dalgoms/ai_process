# 솔루션 — AI 건강 비서

## 도구에서 동반자로

AI 건강 비서는 헬스케어 앱의 역할을 **수동적인 도구**에서 **능동적인 일상 동반자**로 전환하는 AI 건강 어시스턴트입니다.

사용자가 앱을 열기를 기다리는 대신, AI 건강 비서가 먼저 다가갑니다 — 사용자의 데이터, 행동, 패턴을 기반으로 개인화된 건강 인사이트를 전달합니다.

## 작동 방식

```
사용자 건강 데이터  →  AI 분석  →  일일 인사이트  →  사용자 행동
```

1. **수집** — 연동된 소스(웨어러블, 수동 기록, 앱 활동)에서 건강 신호를 수집합니다.
2. **분석** — AI가 수면, 활동, 약물, 바이탈, 식단 등의 패턴을 처리합니다.
3. **전달** — 개인화된 인사이트가 적절한 시점(아침, 저녁)에 사용자에게 전달됩니다.
4. **참여** — 사용자는 건강 타임라인을 통해 자신의 건강 이야기가 펼쳐지는 것을 확인합니다.

## 기존 앱과의 차이

| 기존 헬스케어 앱 | AI 건강 비서 |
|---|---|
| 아플 때 사용자가 앱을 연다 | 매일 앱이 사용자를 찾아간다 |
| 기능 중심 네비게이션 | 인사이트 중심 경험 |
| 정적 대시보드 | 살아있는 건강 타임라인 |
| 일반적인 푸시 알림 | AI 개인화 건강 넛지 |
| 데이터가 방치된다 | 데이터가 매일 가치를 만든다 |

## 핵심 아이디어

> "건강한 날에도 건강은 멈추지 않습니다. 건강 앱도 마찬가지여야 합니다."

AI 건강 비서는 건강 데이터를 **매일 유용하게** 만듭니다 — 아픈 날만이 아니라. 흩어진 데이터 포인트를 사용자가 보고, 이해하고, 행동할 수 있는 하나의 건강 이야기로 만들어줍니다.

## 나만의닥터 수익 구조와의 연결

AI 건강 비서는 독립 수익원이 아닙니다. 기존 수익원의 **전환율을 높이는 성장 엔진**입니다.

```
AI 건강 비서 (무료 참여 레이어)
    │
    ├── 이상 징후 감지 ──→ 비대면 진료 예약 ──→ 진료 수수료
    │
    ├── 약물 리필 알림 ──→ 약 배달 주문 ──→ 배달 수수료
    │
    ├── 건강 콘텐츠 개인화 ──→ healthLab 재방문 ──→ 체류 시간 + 광고 수익
    │
    └── 건강 데이터 축적 ──→ B2B 파트너십 (보험사, 제약사) ──→ 데이터 수익 (향후)
```

핵심은 **매일 앱을 여는 사용자**를 만드는 것입니다. DAU가 올라가면 모든 수익 지표가 함께 올라갑니다.

## 의료 데이터 컴플라이언스

건강 데이터를 다루는 서비스는 반드시 법적 기준을 충족해야 합니다.

| 영역 | 기준 | AI 건강 비서 설계 원칙 |
|---|---|---|
| 개인정보보호법 | 민감정보(건강 데이터) 처리 시 별도 동의 | 온보딩 시 명시적 동의 + 데이터 수집 범위 최소화 |
| 의료법 | AI가 진단/처방을 할 수 없음 | AI 건강 비서는 **정보 제공 및 안내** 역할만 수행, 의료 행위 미포함 |
| 의료기기법 | SaMD(Software as Medical Device) 해당 여부 | 건강 관리 목적의 웰니스 기능으로 설계, 진단/치료 기능 미포함 |
| 데이터 보안 | 건강 데이터 암호화 및 접근 제어 | 전송 시 TLS, 저장 시 AES-256 암호화, 최소 권한 원칙 |

**명확한 경계:** AI 건강 비서는 "의사"가 아니라 "건강 비서"입니다. 증상에 대해 정보를 제공하되, 진단이나 처방은 항상 나만의닥터의 비대면 진료(의사)로 연결합니다.

---

*AI 건강 비서는 진단 도구가 아닙니다. 사용자가 자신의 건강에 관심을 갖고 참여할 수 있도록 돕는 일상 건강 동반자이며, 의료 행위가 필요한 순간에는 나만의닥터의 전문 의료 서비스로 안내합니다.*
